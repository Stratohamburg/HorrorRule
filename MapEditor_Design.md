# 地图编辑器详细设计文档

## 1. 引言
### 1.1 编写目的
本文档旨在详细描述《HorrorRule》游戏地图编辑器的架构设计、核心功能、数据结构及用户界面，为开发人员提供明确的实现指南，确保地图构建工具的高效性、易用性和可扩展性。

### 1.2 适用范围
适用于参与地图编辑器开发的客户端程序、后端程序（若涉及云端存储）、游戏策划及美术人员。

---

## 2. 总体架构设计
地图编辑器主要由以下四个核心模块组成，采用低耦合的模块化设计：

- **数据管理模块 (Data Manager)**：负责地图数据的序列化与反序列化（保存/加载），以及内存中的数据状态维护。
- **渲染模块 (Render Engine)**：负责在编辑器视口中实时绘制地形、物件、网格线、碰撞体及触发器区域。
- **工具与逻辑模块 (Tool & Logic)**：提供笔刷、橡皮擦、填充、选择、撤销/重做（Undo/Redo）等核心编辑逻辑。
- **UI交互模块 (UI System)**：处理用户的鼠标/键盘输入，管理面板（属性面板、资源面板、图层面板等）的显示与交互。

---

## 3. 数据结构设计
为了保证数据的通用性与可读性，地图数据建议采用 JSON 格式进行存储（发布时可转为二进制格式以压缩体积）。

### 3.1 地图基础数据 (MapData)
- `MapID`: string (地图唯一标识)
- `MapName`: string (地图名称)
- `Width`: int (地图宽度/网格数)
- `Height`: int (地图高度/网格数)
- `GridSize`: float (单个网格的物理尺寸)
- `Layers`: List<LayerData> (图层列表)

### 3.2 图层数据 (LayerData)
- `LayerID`: int
- `LayerName`: string (如：背景层、物件层、碰撞层、逻辑层)
- `IsVisible`: bool (是否在编辑器中可见)
- `IsLocked`: bool (是否锁定，锁定后不可编辑)
- `Elements`: List<ElementData> (该图层包含的元素集合)

### 3.3 元素数据 (ElementData)
- `InstanceID`: string (实例唯一ID)
- `AssetID`: string (关联的资源/预制体ID)
- `Position`: Vector3 (世界坐标)
- `Rotation`: Vector3 (旋转角度)
- `Scale`: Vector3 (缩放比例)
- `CustomProperties`: Dictionary<string, string> (自定义属性，如触发器事件参数、怪物刷新配置、解谜机关状态等)

---

## 4. 核心功能模块

### 4.1 文件与工程操作
- **新建地图**：设定地图基础尺寸和网格大小，初始化默认图层（如 Default, Collision, Logic）。
- **保存/另存为**：将内存中的 `MapData` 序列化并写入本地磁盘。
- **加载地图**：解析地图文件，重建内存数据结构，并通知渲染模块刷新视口。

### 4.2 视图控制
- **平移与缩放**：支持鼠标中键平移视口，滚轮缩放。
- **网格对齐 (Snap to Grid)**：开启后，物件的移动和放置将自动吸附到最近的网格交点或中心，确保拼接严丝合缝。
- **辅助线显示 (Gizmos)**：可切换显示/隐藏网格线、碰撞边界、触发器范围、出生点标识等。

### 4.3 编辑工具
- **选择工具 (Select)**：点击选中单个物件，支持拖拽框选多个物件。选中后可在属性面板修改参数。
- **变换工具 (Transform)**：通过操作杆 (Gizmos) 直观地调整物件的位移、旋转和缩放。
- **笔刷工具 (Brush)**：用于快速连续放置地形块或物件，支持设置笔刷大小。
- **橡皮擦工具 (Eraser)**：移除点击或涂抹区域的物件。

### 4.4 撤销/重做系统 (Undo/Redo)
- 采用**命令模式（Command Pattern）**封装每一次修改操作（如 `AddObjectCommand`, `MoveObjectCommand`, `DeleteCommand`）。
- 维护一个历史操作栈，支持 `Ctrl+Z` 撤销和 `Ctrl+Y` 重做，提升容错率。

## 5. 用户界面 (UI) 布局
编辑器界面建议采用经典的停靠式布局（Docking Layout），主要划分为以下几个区域：

1. **顶部菜单栏 (Menu Bar)**：包含 文件(File)、编辑(Edit)、视图(View)、工具(Tools)、帮助(Help)。
2. **快捷工具栏 (Toolbar)**：放置高频操作按钮，如保存、撤销/重做、基础工具（选择、移动、笔刷等）、网格对齐开关、运行测试(Play)。
3. **左侧面板 (Hierarchy & Layers)**：
   - **图层管理器**：管理图层的层级顺序、可见性和锁定状态。
   - **场景大纲**：树状显示当前地图中的所有物件，支持快速定位。
4. **中央视口 (Main Viewport)**：地图的可视化编辑与预览区域。
5. **右侧面板 (Inspector/Properties)**：显示并编辑当前选中物件的详细属性（Transform、自定义逻辑参数等）。
6. **底部面板 (Asset Browser)**：资源浏览器，展示可用的地形贴图、模型预制体、逻辑触发器等，支持直接拖拽到中央视口中进行实例化。

---

## 6. 扩展性与性能优化

### 6.1 性能优化
- **数据分块 (Chunk System)**：对于超大无缝地图，采用九宫格分块加载和保存机制，避免内存溢出和卡顿。
- **视锥体剔除 (Frustum Culling)**：渲染模块仅绘制当前视口可见范围内的物件，保证编辑器在处理复杂场景时的帧率。

### 6.2 扩展性设计
- **插件化架构**：自定义属性面板支持反射或接口扩展。当策划需要为特定游戏逻辑（如恐怖游戏中的“惊吓触发器”、“灯光闪烁控制器”）增加专属编辑UI时，程序只需编写对应的属性绘制脚本，无需修改编辑器核心代码。
- **数据导出管线**：支持编写自定义导出脚本，将编辑器的原始 JSON 数据一键打包、烘焙为游戏引擎直接读取的优化格式。

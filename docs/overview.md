# 《恐怖规则》游戏设计概览文档

## 一、游戏概述

### 1.1 游戏名称
- 中文名：恐怖规则 / 规则怪谈
- 英文名：Horror Rule

### 1.2 游戏类型
- 2D 俯视角恐怖解谜生存游戏

### 1.3 游戏平台
- Web端（移动端/PC端浏览器均可运行）

### 1.4 技术方案
- 纯前端实现，基于 HTML5 Canvas
- 无后端服务器依赖
- 游戏引擎：Phaser.js 或原生 Canvas 2D

### 1.5 美术风格
- 像素风格（类似 RPG Maker / 经典像素恐怖游戏）
- 暗色调为主，营造压抑恐怖氛围
- 有限视野照明系统（手电筒/烛光效果）

### 1.6 游戏模式
- 单人游戏
- 关卡制，每关对应不同场景和规则怪谈主题

### 1.7 核心玩法一句话描述
> 玩家在充满诡异规则的场景中探索，通过阅读、理解并遵守（或打破）规则来生存下去。违反真规则将遭遇恐怖事件甚至死亡，识别并无视假规则才能找到真正的逃生之路。

## 二、操作与视角

### 2.1 视角
- 第三人称俯视角（Top-Down View）
- 摄像机跟随玩家角色移动，角色始终处于画面中央
- 视野范围受限（迷雾/黑暗系统），仅能看到角色周围一定范围

### 2.2 虚拟摇杆
- 位置：屏幕中下方居中
- 样式：半透明圆形底座 + 可拖动的操控点
- 底座直径：约 120px（可根据屏幕自适应缩放）
- 操控点直径：约 50px
- 触控区域：底座范围内任意位置点击即可激活摇杆
- 操控方式：
  - 触摸/鼠标按住底座区域并拖动来控制方向
  - 支持360°全方向移动
  - 拖动距离决定移动速度（小幅拖动=慢走，大幅拖动=快走）
  - 松手后角色停止移动，摇杆回到中心位置
- 透明度：未触控时 40% 透明度，触控时 80% 透明度

### 2.3 交互操作
- **交互按钮**：屏幕右下方设置一个"交互"按钮（手形图标）
  - 靠近可交互物体时按钮高亮
  - 点击后触发交互（拾取物品、阅读文字、开门等）
- **背包按钮**：屏幕右上方设置背包图标
  - 点击打开背包界面，查看已获得的物品和已发现的规则
- **规则手册按钮**：屏幕右上方设置书本图标（紧邻背包按钮）
  - 点击打开当前关卡已发现的所有规则列表
  - 玩家可以标记规则为"真"或"假"（标记仅为辅助记忆，不影响游戏逻辑）
- **暂停按钮**：屏幕左上角齿轮图标

### 2.4 PC端适配
- 支持 WASD / 方向键移动
- 支持鼠标点击交互
- 空格键作为交互键
- Tab键打开背包
- R键打开规则手册

## 三、核心玩法机制

> 📖 以下为核心玩法的概要说明。所有系统的**详细数值、判定逻辑和数据结构**请参见 [核心规则汇总文档 (core_rule.md)](core_rule.md)。

### 3.1 规则系统

规则怪谈是本游戏的核心。每个关卡中，玩家会通过各种方式获得"规则"（基础规则手册、场景中的告示、NPC对话等），并需要自行判断其真伪来决定是遵守还是无视。

规则分为五大类：**真规则**、**假规则**、**矛盾规则**、**条件规则**、**陷阱规则**。玩家需通过视觉差异、逻辑冲突、环境暗示、编号异常等线索来辨别。

### 3.2 理智值系统（SAN值）

理智值是玩家的"精神生命值"（上限100）。目击恐怖事件、违反真规则、处于黑暗中等会降低SAN值；使用道具、处于安全区域、识破假规则可恢复SAN值。SAN值归零则游戏结束。

不同SAN值区间会产生递进的负面效果，从轻微的画面异常到严重的幻觉和移速下降。

### 3.3 体力系统（Stamina）

玩家拥有体力值（上限100），奔跑会持续消耗体力。体力耗尽后进入"力竭"状态，移速大幅下降且无法再次奔跑，直到体力恢复到一定阈值。这为怪物追逐中的逃跑策略增加了资源管理维度。

### 3.4 探索与交互

场景中包含多种可交互物体（门、纸条、抽屉、开关、NPC、藏身点等）。交互需要玩家在一定距离内且面朝物体方向。交互分为瞬间交互（如拾取）和读条交互（如撬锁、躲藏）两种。

### 3.5 道具与背包系统

背包容量有限（8格），道具分为三类：
- **关键道具（KeyItem）**：钥匙、任务物品等，不可丢弃
- **消耗品（Consumable）**：镇定药片、电池、火柴等，可叠加
- **线索（Clue）**：纸条、日记等，拾取后进入规则手册/线索库，不占用背包格

### 3.6 怪物/异常实体系统

怪物拥有视觉和听觉感知，通过警戒值系统在 巡逻→警觉→追逐→搜索 状态间转换。不同类型的怪物（巡逻型、追逐型、静止型、模仿型、条件触发型）有不同的行为逻辑。玩家的奔跑、开门等动作会产生噪音，增加被发现概率。

### 3.7 视野与光照系统

游戏采用 Shadow Casting 算法实现动态光照。玩家视野受限于光源范围，不同光源（基础视野、手电筒、蜡烛）有不同的形状和参数。长时间处于黑暗中会影响SAN值，同时光源也会增加玩家被怪物发现的概率，形成安全感与暴露风险的博弈。

## 四、关卡设计

### 4.1 关卡通用结构

每个关卡遵循以下通用结构：
1. **开场**：玩家进入场景，获得初始规则手册
2. **探索期**：自由探索场景，收集线索和道具
3. **升级期**：场景发生变化（灯光变暗、出现新区域、怪物开始活动）
4. **抉择期**：需要根据规则做出关键选择
5. **逃生/结算**：完成目标后找到出口逃生

| 第二关 | 蓝海市极地海洋馆 | 封闭场馆 | 大型 | 深海恐惧、被注视感 |

## 六、音频设计

### 6.1 背景音乐
- 主菜单：低沉的钢琴旋律+环境音
- 探索阶段：极简的氛围音乐（几乎无旋律，以环境声为主）
- 紧张阶段：心跳声加速+不和谐弦乐
- 追逐阶段：急促的打击乐+尖锐弦乐
- 安全区域：轻柔的钢琴或音乐盒旋律

### 6.2 音效
| 音效类型 | 说明 |
|----------|------|
| 脚步声 | 根据地面材质不同（木地板/瓷砖/地毯） |
| 门声 | 开门/关门/锁门/门被撞击 |
| 环境音 | 风声/雨声/时钟滴答/水管滴水 |
| 恐怖音效 | 尖叫/低语/骨骼碎裂/不明敲击声 |
| 交互音效 | 拾取物品/翻页声/开关声 |
| UI音效 | 按钮点击/菜单打开关闭 |

### 6.3 音频策略
- 利用"安静-突然"对比制造恐怖效果（Jump Scare 音效）
- 空间音效：声音有方向性，暗示危险方向
- 低频嗡鸣声：在异常区域持续播放，制造不安感

### 7.3 关键算法

> 📖 以下算法的详细参数、数值和实现逻辑请参见 [核心规则汇总文档 (core_rule.md)](core_rule.md)。

- **视野/光照系统**：基于 Shadow Casting 算法的 2D 光线投射，支持圆形/扇形光源，经墙壁遮挡计算
- **怪物AI**：A* 寻路 + 路点巡逻 + 视锥/声音检测 + 警戒值驱动的状态机（巡逻→警觉→追逐→搜索）
- **规则引擎**：基于事件驱动的数据化规则检测系统，每条规则绑定触发器(Trigger)、条件(Condition)和结果(Action)

## 八、存档系统

### 8.1 自动存档
- 每进入一个新区域/触发关键事件时自动保存
- 使用浏览器 LocalStorage 存储
- 存储数据包括：
  - 当前关卡ID
  - 玩家位置坐标
  - 理智值
  - 背包物品
  - 已发现的规则列表
  - 已触发的事件标志
  - 游戏内时间
  - 怪物状态

## 九、性能与兼容性

### 9.1 目标性能
- 稳定 60FPS（移动端至少30FPS）
- 首屏加载时间 < 3秒
- 单关卡资源总量 < 5MB

### 9.2 兼容性
- Chrome 80+、Firefox 75+、Safari 13+、Edge 80+
- 移动端：iOS Safari、Android Chrome
- 最低分辨率支持：375×667（iPhone SE）
- 推荐分辨率：414×896 / 1920×1080

### 9.3 优化策略
- 精灵图合并（Sprite Atlas）减少绘制调用
- 只渲染视口内的对象（Camera Culling）
- 音频按需加载/卸载
- 地图分块加载（大型关卡）

## 十一、设计原则

### 11.1 恐怖设计哲学
1. **暗示比展示更恐怖**：少用Jump Scare，多用环境叙事和暗示
2. **未知是最大的恐惧**：不完整的信息、自相矛盾的规则、不可预测的后果
3. **规则感与失控感的对比**：规则给予安全感，规则被打破/发现规则是假的时产生强烈不安
4. **玩家的选择带来恐惧**：不确定自己是否做对了，这种纠结本身就是恐怖体验

### 11.2 游戏设计原则
1. **公平性**：每个假规则都有线索可以辨别，不做无理要求
2. **渐进难度**：前几关规则简单直接，后面越来越复杂
3. **多次游玩价值**：每关有隐藏线索和多结局，鼓励重复探索
4. **即时反馈**：违反规则后立刻有反馈（音效、画面变化、SAN值下降）

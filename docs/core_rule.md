# 《恐怖规则》核心规则汇总文档

本文档旨在集中汇总《恐怖规则》（Horror Rule）游戏中最核心的底层规则与系统机制，作为关卡设计、程序开发和玩家引导的基础准则。

---

## 一、 规则怪谈系统 (The Rule System)

规则是本游戏的核心驱动力。玩家的生存完全依赖于对规则的获取、理解、辨别和执行。

### 1.1 规则的分类
游戏中的规则对玩家隐藏其真实属性，玩家必须自行判断。规则分为以下五类：

| 类型 | 核心特征 | 应对策略 | 违反/执行后果 |
| :--- | :--- | :--- | :--- |
| **真规则** | 维持场景平衡的底层法则，通常由“官方”或“善意实体”留下。 | **必须严格遵守**。 | 触发恐怖事件、扣除大量SAN值、引来怪物，严重时直接死亡。 |
| **假规则** | 由“恶意实体”或“怪物”伪造，目的是诱导玩家打破平衡。 | **必须无视或反向操作**。 | 遵守假规则会直接落入陷阱，导致受伤或死亡；成功识破并无视可恢复少量SAN值。 |
| **矛盾规则** | 两条或多条规则在逻辑上完全冲突（如：A说往左，B说往右）。 | **寻找第三方线索**（环境暗示、道具、NPC）来印证其中一条的真实性。 | 遵守错误的一方等同于违反真规则。 |
| **条件规则** | 仅在特定时间、特定地点或特定状态下才生效的规则。 | **确认当前触发条件**。在非生效条件下执行通常无害但也无益。 | 在生效条件下违反，等同于违反真规则。 |
| **陷阱规则** | 伪装成普通交互提示的规则（如“按下红色按钮”）。 | **结合环境常识判断**，切勿盲目交互。 | 一旦执行，立即触发负面事件（如停电、锁门、刷怪）。 |

### 1.2 规则的辨别线索
玩家可以通过以下维度来辨别规则的真伪：
*   **视觉差异**：字迹潦草、字体颜色异常（如暗红色）、纸张破损、被涂改的痕迹。
*   **逻辑冲突**：与常识严重违背，或与已验证的“真规则”产生直接冲突。
*   **环境暗示**：规则要求去某个房间，但该房间门口有大量血迹或警告标志。
*   **编号异常**：规则列表中突然出现乱码编号，或者如“规则3.5”、“第13条”（当总数只有12条时）。

## 二、 理智值系统 (SAN System)

理智值（SAN）是玩家的“精神生命值”，归零即意味着游戏结束。

### 2.1 SAN值基础设定
*   **上限**：100
*   **UI表现**：屏幕上方的“眼睛”图标。随着SAN值降低，眼睛会逐渐充血、半闭。

### 2.2 SAN值消耗与恢复
**扣除途径（精神污染）：**
*   目击恐怖事件/怪物真容：-5 ~ -20（单次）
*   违反真规则（未致死情况下）：-10 ~ -30（单次）
*   处于完全黑暗环境：-2 / 每30秒
*   被怪物追逐：-1 / 每秒
*   阅读受污染的禁忌文本：-5 ~ -15（单次）

**恢复途径（精神锚点）：**
*   处于绝对安全区域（如明亮的休息室）：+1 / 每5秒（最高恢复至80）
*   使用消耗品（镇定药片、特定饮料）：+15 ~ +30（单次）
*   成功识破并无视一条假规则：+5（单次，给予正向反馈）

### 2.3 SAN值阶段性负面效果
| SAN值区间 | 状态描述 | 游戏内表现（视觉/听觉/玩法） |
| :--- | :--- | :--- |
| **100 - 70** | 清醒 | 视野清晰，无异常。 |
| **69 - 40** | 焦虑 | 屏幕边缘偶现暗红色暗角；偶尔听到不存在的低语或脚步声。 |
| **39 - 20** | 幻觉 | 画面轻微扭曲（水波纹效果）；场景中出现假怪物阴影；规则文本可能出现短暂乱码。 |
| **19 - 1** | 崩溃边缘 | 画面严重扭曲、高频闪烁；移动速度降低20%；无法准确阅读新规则（文字模糊）。 |
| **0** | 彻底疯狂 | 角色失去控制，被黑暗吞噬，**Game Over**。 |

## 三、 实体与感知系统 (Entity & Perception)

游戏中的威胁主要来自于各种异常实体（怪物），它们遵循特定的感知逻辑。

### 3.1 玩家的隐蔽与暴露（噪音与视线）
*   **噪音机制**：
    *   潜行（蹲走/慢走）：噪音范围极小。
    *   正常行走：中等噪音范围。
    *   奔跑：大范围噪音，极易惊动听觉敏锐的实体。
    *   交互噪音：开门、打碎物品、砸镜子会产生瞬间的高分贝噪音源。
*   **光源暴露**：开启手电筒会大幅增加玩家的可见度，容易被远处的视觉型实体发现。

### 3.2 实体的行为逻辑
*   **巡逻型**：按固定路线移动，拥有锥形视野。玩家需利用掩体和阴影躲避。
*   **追逐型**：一旦发现玩家（视觉或听觉触发），会持续追击，直到玩家逃入“安全屋”或躲进“藏身点”（如床底、柜子）并丢失视野。
*   **机制/静止型**：不主动移动，通常作为“规则惩罚”存在。只有玩家违反特定规则（如回头、直视、发出声音）时才会发动致命攻击。
*   **模仿型**：伪装成正常NPC或场景物品。靠近或交互时会暴露真面目并造成巨额SAN值打击。

## 四、 核心生存法则 (给玩家的暗语)

1.  **不要相信你看到的一切**：文字可能是伪造的，NPC可能是伪装的，甚至你的眼睛（低SAN值时）也会欺骗你。
2.  **保持理智是第一职业道德**：黑暗和未知会吞噬你的理智，永远保证手电筒有电，或者知道下一个光源在哪里。
3.  **犹豫就会败北，盲动就会白给**：遇到新规则时，先观察环境再行动；但当怪物追来时，立刻寻找藏身处。
4.  **规则既是枷锁，也是护身符**：真规则虽然苛刻，但只要严格遵守，即使是再恐怖的实体也无法伤害你。

## 五、 角色控制与体力系统 (Player Control & Stamina)

为了增加生存压力，引入体力（Stamina）系统，限制玩家的无限奔跑。

### 5.1 基础数值设定
*   **基础移动速度（Walk）**：100 px/s
*   **奔跑移动速度（Run）**：220 px/s
*   **潜行移动速度（Crouch）**：50 px/s
*   **体力值上限（Max Stamina）**：100
*   **体力消耗（奔跑）**：-15 / 秒
*   **体力恢复（静止/行走/潜行）**：+10 / 秒（奔跑停止后延迟 1.5 秒开始恢复）

### 5.2 疲劳状态（Exhaustion）
*   当体力值降至 **0** 时，触发【力竭】状态。
*   **力竭惩罚**：强制切换为“喘气行走”状态，移动速度降至 70 px/s，且在此期间无法再次奔跑。
*   **解除条件**：体力值自动恢复至 **50** 以上时，解除力竭状态。
*   **UI表现**：详见 `interface_rule.md` 中体力条（Stamina Bar）的显示规范。

## 六、 视野与光照计算 (Vision & Lighting)

光照不仅是视觉效果，更是核心的玩法判定机制（影响SAN值和怪物仇恨）。

### 6.1 光源类型与参数
| 光源类型 | 形状 | 半径/距离 | 角度 | 衰减边缘 | 消耗 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **玩家基础视野** | 圆形 | 60 px | 360° | 10 px | 无 |
| **手电筒（开启）** | 扇形 | 350 px | 65° | 30 px | 电量 -1/秒 (满电100) |
| **蜡烛/火柴** | 圆形 | 150 px | 360° | 20 px | 持续 30 秒后熄灭 |
| **环境灯光(路灯/顶灯)**| 圆形/多边形 | 视场景而定 | 360° | 视场景而定 | 无（受开关/停电事件控制） |

### 6.2 “黑暗状态”的程序判定逻辑
游戏需要每秒检测玩家是否处于“黑暗”中，以决定是否扣除SAN值或触发特定怪物。
*   **判定算法**：
    1. 获取玩家当前中心点坐标 `(Px, Py)`。
    2. 遍历当前场景中所有**激活状态**的光源。
    3. 如果 `(Px, Py)` 不在任何一个光源的有效多边形（经过墙壁遮挡计算后的Shadow Cast多边形）内，则判定玩家处于【黑暗状态】。
    4. 处于黑暗状态累计超过 **3秒**，开始执行SAN值扣除逻辑（-2/每30秒，或触发“黑暗中的低语”事件）。

## 七、 规则判定引擎架构 (Rule Evaluation Engine)

这是游戏最核心的代码逻辑。规则不能硬编码，必须数据化，以便关卡设计师配置。

### 7.1 规则数据结构 (JSON 示例)
每条规则在代码中应表现为一个包含“触发器(Trigger)”、“条件(Condition)”和“结果(Action)”的对象。

```json
{
  "ruleId": "R_<LevelId>_<Seq>",
  "description": "<规则的文本描述，将展示给玩家>",
  "type": "TRUE_RULE | FALSE_RULE | CONDITIONAL_RULE | TRAP_RULE",
  "triggerEvent": "ON_INTERACT | ON_MOVE | ON_LOOK_AT | ON_TIME_REACHED | ON_ITEM_USED",
  "targetEntity": "<场景中可交互物体的ID>",
  "conditions": [
    { "check": "IS_EVENT_ACTIVE", "eventId": "<事件ID>", "value": true }
  ],
  "onViolate": [
    { "action": "PLAY_SOUND", "soundId": "<音效ID>" },
    { "action": "MODIFY_SANITY", "value": -40 },
    { "action": "SPAWN_MONSTER", "monsterId": "<怪物ID>", "x": 0, "y": 0 },
    { "action": "SET_GAME_OVER", "delay": 2.0, "reason": "<死亡原因文本>" }
  ],
  "onObey": [
    { "action": "WAIT_TIME", "value": 10.0 },
    { "action": "STOP_EVENT", "eventId": "<事件ID>" }
  ]
}
```
> **注意**：以上为通用模板，具体的 `ruleId`、`targetEntity`、`monsterId` 等字段值由各关卡配置文件定义，详见 `levels.md`。

### 7.2 核心事件监听器 (Event Listener)
程序需要全局监听以下玩家行为，并将其抛给规则引擎进行比对：
*   `ON_MOVE` (移动到特定区域/坐标)
*   `ON_INTERACT` (与特定物体交互)
*   `ON_LOOK_AT` (视线/手电筒照射到特定实体，持续X秒)
*   `ON_TIME_REACHED` (游戏内时间到达某一点)
*   `ON_ITEM_USED` (在特定地点使用了特定物品)

## 八、 怪物AI状态机 (Monster AI State Machine)

怪物的行为必须可预测且有逻辑，给玩家提供博弈空间。

### 8.1 警戒值系统 (Suspicion Meter)
怪物发现玩家不是瞬间的（除非贴脸），而是基于警戒值（0-100）。
*   **视觉增加**：玩家在怪物视野锥内。距离越近、玩家处于光源下，警戒值增加越快（+20~50/秒）。
*   **听觉增加**：玩家在怪物听觉半径内奔跑（+30/秒）或制造巨大噪音（瞬间+80）。
*   **衰减**：玩家脱离感知范围后，警戒值 -10/秒。

### 8.2 AI 状态转换逻辑
1.  **【巡逻状态 Patrol】**：
    *   行为：在预设的路点（Waypoints）之间以 80 px/s 的速度移动。
    *   转换：警戒值 > 0 时，进入【警觉状态】。
2.  **【警觉状态 Suspicious】**：
    *   行为：停止巡逻，面朝噪音/视觉来源方向，播放低吼音效。如果警戒值继续增加，缓慢走向目标点。
    *   转换：警戒值达到 100，进入【追逐状态】；警戒值降回 0，返回【巡逻状态】。
3.  **【追逐状态 Chase】**：
    *   行为：播放追逐BGM，以 200 px/s 的速度（略低于玩家奔跑速度）使用 A* 寻路算法追击玩家。
    *   转换：玩家进入捉迷藏点或脱离怪物视野超过 5 秒，进入【搜索状态】；追上玩家（距离 < 30px），触发【攻击/处决】。
4.  **【搜索状态 Search】**：
    *   行为：在玩家最后消失的坐标附近 150px 范围内随机游荡 10 秒。
    *   转换：10秒后未发现玩家，警戒值清零，返回【巡逻状态】。

## 九、 物品与交互系统 (Inventory & Interaction)

### 9.1 背包设定
*   **容量限制**：固定 8 个格子（增加资源管理压力）。
*   **物品分类**：
    *   `KeyItem`（关键道具）：钥匙、门禁卡、任务物品（不可丢弃）。
    *   `Consumable`（消耗品）：镇定药片、电池、火柴（可叠加，上限5）。
    *   `Clue`（线索）：纸条、日记（拾取后直接进入“规则手册/线索库”，**不占用背包格子**）。

### 9.2 交互判定 (Raycast / Overlap)
*   **触发范围**：玩家中心点半径 40 px 内。
*   **朝向判定**：玩家的面朝向向量与物体中心点的夹角必须小于 45°，才会显示【交互提示】（避免背对物体也能交互的违和感）。
*   **交互类型**：
    *   **瞬间交互**：点击立刻生效（如捡起钥匙、开普通门）。
    *   **读条交互**：按住交互键持续 X 秒（如撬锁、躲进床底、解除陷阱）。读条期间移动或受到攻击会打断进度。

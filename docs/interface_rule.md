# 《恐怖规则》界面与交互规范文档 (Interface & UI Rule)

本文档详细规定了《恐怖规则》游戏中的所有UI界面、界面元素构成、以及各界面之间的跳转逻辑与层级关系。

---

## 一、 界面拓扑图与层级关系 (UI Flow & Hierarchy)

### 1.1 界面跳转拓扑图
```text
[启动游戏]
   │
   ▼
【主菜单界面】(Main Menu) ───► 【设置界面】(Settings)
   │
   ▼
【关卡选择界面】(Level Select)
   │
   ▼
【游戏主界面 / HUD】(Game HUD) ◄────────┐
   │                                    │
   ├─► 【规则手册界面】(Rule Book) ─────┤ (覆盖在HUD之上，游戏暂停/半暂停)
   ├─► 【背包界面】(Inventory) ─────────┤
   ├─► 【对话/阅读界面】(Dialogue) ─────┤
   ├─► 【暂停界面】(Pause Menu) ────────┘
   │
   ├─► 【死亡界面】(Game Over) ───┬──► (重试) ──► 【游戏主界面】
   │                              └──► (退出) ──► 【主菜单界面】
   │
   └─► 【通关结算界面】(Victory) ─────► (下一关/返回) ──► 【关卡选择界面】
```

### 1.2 UI 渲染层级 (Z-Index)
在前端渲染引擎（如 Phaser.js）中，UI的层级从底到高依次为：
1. `Z-0`: 游戏场景渲染层 (地图、角色、怪物)
2. `Z-1`: 光照与阴影遮罩层 (Shadow/Fog of War)
3. `Z-2`: 屏幕特效层 (低SAN值时的血丝、屏幕扭曲、CRT扫描线)
4. `Z-3`: 基础 HUD 层 (摇杆、交互按钮、SAN值显示)
5. `Z-4`: 交互弹窗层 (对话框、阅读纸条)
6. `Z-5`: 全屏系统界面层 (背包、规则手册、暂停菜单)
7. `Z-6`: 顶层转场动画层 (黑屏淡入淡出、死亡Jump Scare闪图)

## 二、 核心界面详细说明

### 2.1 主菜单界面 (Main Menu)
*   **视觉风格**：极简、压抑。背景为动态的微光像素图（如一扇半掩的门，透出微弱红光）。
*   **核心元素**：
    *   **游戏Logo**：居中偏上，带有轻微的故障（Glitch）闪烁特效。
    *   **菜单选项**：居中偏下，包含 `[开始游戏/继续游戏]`、`[关卡选择]`、`[设置]`、`[关于]`。
    *   **交互反馈**：鼠标悬停或手指触摸时，选项文字变为血红色，并伴随低沉的“滴答”音效。

### 2.2 关卡选择界面 (Level Select)
*   **布局**：横向滑动的卡片式布局。
*   **卡片内容**：
    *   关卡缩略图（未解锁为黑白噪点图，已解锁为场景截图）。
    *   关卡名称（如：第一关 - 深夜便利店）。
    *   最高评级（S/A/B/C）及收集进度（如：线索 3/5）。
    *   锁图标（针对未解锁关卡）。

### 2.3 游戏主界面 / HUD (Game HUD)
这是玩家停留时间最长的界面，设计原则是**“尽量减少遮挡，突出沉浸感”**。

*   **顶部区域**：
    *   **左上角**：`[暂停]` 按钮（齿轮图标）。
    *   **中上方**：`[SAN值监视器]`。一个像素风格的"眼睛"图标 + 数字（如 85/100）。眼睛的状态会随SAN值变化（睁开 -> 半闭 -> 充血 -> 疯狂转动）。
    *   **SAN值下方**：`[体力条]`（Stamina Bar）。窄长条形式，平时隐藏；体力不满或正在消耗时自动显示，满值后延迟2秒消失。体力耗尽（力竭状态）时体力条闪烁红光。
    *   **右上角**：`[背包]` 按钮（背包图标）和 `[规则手册]` 按钮（书本图标）。当有新规则或新物品时，图标右上角显示红点提示。
*   **中部区域**：
    *   **底部居中偏上**：`[游戏内时间]`（如 "AM 2:30"）。仅在有时间机制的关卡中显示。半透明小字，不遮挡视野。在关卡时间限制临近时，文字变为红色闪烁并增大字号以警示玩家。
*   **底部区域 (移动端特有)**：
    *   **左下角**：`[虚拟摇杆]`。半透明，动态呼出（手指按下时以触点为中心生成摇杆底座）。
    *   **右下角**：`[交互/动作按钮]`。平时为半透明隐藏状态；当靠近可交互物体时，高亮显示为"手形"或"放大镜"图标。
*   **PC端适配**：隐藏虚拟摇杆和交互按钮，在屏幕右下角显示半透明的按键提示（如 `[Space] 交互`, `[Tab] 背包`, `[R] 规则手册`）。

### 2.4 规则手册界面 (Rule Book)
*   **呼出方式**：点击HUD书本图标或按 `R` 键（PC端）。呼出时游戏时间暂停。
*   **视觉风格**：一本破旧的、带有血迹或污渍的笔记本。
*   **核心功能**：
    *   **规则列表**：按发现顺序排列当前关卡收集到的所有规则。
    *   **真伪标记系统**：每条规则右侧有三个单选按钮：`[?] 未知`、`[O] 确信为真`、`[X] 确信为假`。这仅作为玩家的辅助记忆工具，不影响底层逻辑。
    *   **线索附录**：翻页可查看收集到的纸条原文或日记碎片。

### 2.5 背包界面 (Inventory)
*   **呼出方式**：点击HUD背包图标或按 `Tab` 键。呼出时游戏时间暂停。
*   **布局**：
    *   **左侧/上方**：8个物品网格（2x4 或 1x8）。选中物品时格子高亮。
    *   **右侧/下方**：物品详情区。显示物品放大图标、名称、描述文本。
    *   **操作按钮**：根据物品类型显示 `[使用]`、`[装备]` 或 `[丢弃]`（关键道具不可丢弃）。

### 2.6 交互与对话界面 (Dialogue & Reading)
*   **对话框 (NPC交互)**：
    *   位于屏幕底部，占据约 1/3 高度。
    *   左侧显示NPC像素头像（根据剧情可能发生诡异变化）。
    *   右侧为打字机效果输出的文本。
    *   右下角有向下的小箭头提示点击继续。
*   **阅读界面 (查看纸条/告示)**：
    *   屏幕中央弹出半透明的纸张背景。
    *   文字使用手写体或打印机字体。
    *   点击屏幕边缘任意空白处关闭。

### 2.7 死亡与结算界面 (Game Over & Victory)
*   **死亡界面 (Game Over)**：
    *   **转场**：瞬间的 Jump Scare 闪图或屏幕剧烈撕裂特效，随后陷入纯黑。
    *   **文字**：血红色的“你已被吞噬”或“理智归零”。
    *   **死因提示**：下方用小字显示死因（如：“你违反了规则3：不要直视它的眼睛”），帮助玩家复盘。
    *   **操作**：`[从上个检查点重试]`、`[退出关卡]`。
*   **通关结算界面 (Victory)**：
    *   **视觉**：压抑感解除，色调转为冷白或微暖色。
    *   **数据统计**：显示通关耗时、剩余SAN值、识破的假规则数量、收集的隐藏线索数。
    *   **综合评级**：根据上述数据盖上 S/A/B/C 的印章动画。

## 三、 UI 动效与反馈规范 (UI Animation & Feedback)

为了增强恐怖氛围，UI本身也需要具备“情绪”。

1.  **低SAN值 UI 异变**：
    *   当 SAN < 40 时，HUD 上的文字偶尔会出现乱码（如“背包”瞬间变成“肉块”，持续0.1秒）。
    *   规则手册中的文字可能会自己扭曲、渗血，甚至出现原本不存在的“第13条规则”。
2.  **心跳反馈**：
    *   当怪物靠近或处于极度危险状态时，屏幕边缘的红色暗角会伴随心跳音效进行同频的透明度呼吸闪烁。
3.  **按钮交互**：
    *   所有系统按钮（非游戏内物体）的点击音效不应是清脆的“滴”声，而应是沉闷的纸张翻动声、木头敲击声或低沉的机械咔哒声。
